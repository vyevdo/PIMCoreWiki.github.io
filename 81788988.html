Реклама

  • Реклама заводится в PIMCore, ЕТР, Navision
  • Для рекламы есть отдельный шаблон для загрузки в Navision. Данные для него берутся из заявки поставщика, часть проставляется вручную. Шаблон содержит такие поля:
    • Поля, которые есть в заявке поставщика:
      • ШК
      • ТМ
      • Линия
      • Товарная группа (тип товара)
      • Артикул
      • Код ВЭД
      • Объем
      • Цвет варианта
      • Base unit
      • Название - поле, которое загружается в Navision в отличии от названия, которое Navision формирует самостоятельно по маске для других типов товаров. В будущем для рекламы логика будет такая же, как и для других типов товаров
    • Поля, которые добавляет товаровед:
      • Код товара (код Тетры)
      • Назначение -  всегда Promotional products

Для иерархии 7 - это реклама, 1 - это товар

      • Уровень иерархии 1
      • Уровень иерархии 2
      • Уровень иерархии 3
      • Уровень иерархии 4
      • два поля с ценой - по умолчанию 0 и 1 соответственно

Price-list

  • Прайс-листы вносятся в Тетру через импорт Excel файлов, в Navision информация переносится автоматически, в.ч. информация по Hexagone
  • Информация по прайс-листам приходит сотрудникам от поставщиков
  • Типы цен:
    • цена закупки
    • РРЦ
    • прайс для остатков - ставятся цены РРЦ (дублируются)
    • для поставщиков
  • Для каждой позиции задается цена и дата, с которой цена начинает действовать, информация затягивается по ШК или коду Тетры
  • Файл Excel содержит код товара и ШК, по ним прайс листы можно затянуть в Тетру

Прайс-листы в Navision

Цены в Navision попадают через экспорт из Тетры по установленному алгоритму автоматом

Brocard - прайс-лист для всей розничной сети, также прайсы делятся по некоторым типам магазинов, прайс для остатков, прайс для интернет-магазина Брокард

Справочник Область применения

Внизу указаны все области применения, а также указано, сколько товаров относятся к той или иной области

Справочник Цвет тона

Это общий справочник по всем ТМ

В случае, если появляется новый цвет, которого нет в справочнике, Navision выдаст ошибку при импорте данных. Нужно будет сначала добавить цвет в общий справочник, а потом в Структуру базовых значений, и только после этого Navision позволит залить информацию

Структура базовых значений

Для того, чтоб добавить новый цвет, необходимо в конкретной ТМ марке добавить цвет из общего списка цветов, в который цвет был добавлен выше. Также при наличии номера тона он тоже добавляется
