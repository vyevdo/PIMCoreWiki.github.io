2. Бизнес цели и задания для фазы Development

Оптимизация процессов создания, обновления и использования информации о товаре, за счет создания единого источника достоверной информации о товаре – мастер системы, и интеграции его с другими системами компании. В исключительных случаях допускается возможность изменения данных в мастер системе другими системами (мутация).

Для этого необходимо:

№

Цель

1Ц

Создать единый источник достоверной информации о товаре

2Ц

Обеспечить функционирование программ (систем), использующих товарную номенклатуру, с единым источником достоверной информации

3Ц

Автоматизировать процессы обмена информацией о товарах между системами компании

4Ц

Обеспечить необходимую функциональность систем после вывода ЕТР из эксплуатации

5Ц

Обеспечить возможность ввода информации о товарной номенклатуре поставщиками в полном и в упрощенном виде (без заполнения необязательных атрибутов) с возможностью последующего редактирования

6Ц

Настроить проверку первичной информации о товарах на уровне системы

7Ц

Обеспечить информирование поставщика и товароведов о новой заявке

8Ц

Обеспечить процесс заведения новой номенклатуры

9Ц

Обеспечить процесс внесения изменений в существующую номенклатуру
10ЦПредложить систему импорта сведений о товаре из внешних источников (СSV, API)
11ЦНастроить уведомление круга сотрудников, которые получают информацию о том, что в PimCore обновились/появились новые данные. Уведомления должны поступать с перечнем позиций, названиями таких позиций, брендом и назначением каждой

4.3.2.1 PimCore ↔ Navision

В результате анализа был выбран следующий обмен данными:

  • через запросы

Логика работы через запросы:

Система Navision формирует запрос в PimCore на данные о товаре. PimCore выдает эти данные

Типы запросов:

  • запрос на получение данных 
  • запрос на изменения данных (Используется только для атрибута Navloaded)
Особенности формирования запроса на получение группы продуктов:
  • есть возможность указывать язык для локализации полей
  • есть возможность делать пагинацию 
  • есть возможность делать сортировку
  • есть возможность фильтрации данных
  • есть возможность указывать набор полей для выгрузки

Особенности формирования запроса на получение справочников:

  • доступные поля можно регулировать для каждого потребителя данных
  • для крупных справочников стоит выгружать только измененные данные
  • каждая система может иметь свою точку актуализации данных и получать только актуализированные. Для ее обновления, используя modificationDate - общесистемный атрибут который есть у всех # объектов системы, так же как и PIMCore

4.6.2 Схемы процессов согласно с перечнем

4.6.2.1 Создание уникального идентификатора товара

Для более правильной работы систем был создан уникальный идентификатор: новый атрибут object_id

Данный атрибут является уникальным ID товара либо ID справочника. Его можно использовать вместо Кода Тетра.

Данный идентификатор создается в момент создания товара либо нового элемента справочника в системе PIM Core

4.6.2.2 Валидация данных

Иметь актуальный и единый достоверный источник информации о товаре крайне важно для бизнеса. В связи с этим было реализовано: 

  1. Написана инструкция пользователя с учетом обновленного списка проверок
  2. Формализован список проверок, которые были обновлены в PimCore для поддержания правильного ассортимента
  3. Настроены множество связей между справочниками и товарами
  4. Стандартизировано поле объем
  5. Настроены другие проверки при заведении нового товара. Детальный перечень проверок отображон в файле "Название файла"
  6. Настроено автоматическое заполнение следующих полей в PimCore для упрощения работы поставщика и товароведов:
    1. Категории всех уровней
    2. Область применения Тетра через тип товара
    3. Область применения NAV через тип товара
    4. Линия на украинском языке
    5. Страна происхождения (для существующих брендов)
    6. Производитель (для существующих брендов)
    7. Признак импортного товара
    8. Название товара для PimCore

4.6.2.3 Процесс создания набора

В связи с необходимостью бизнеса формировать набор из существующих товаров самостоятельно, а так же разделять существующий набор на компоненты, важно заводить набор вместе с его комплектующими.

Поскольку набор является товаром, то его необходимо хранить в едином хранилище данных о товаре "PimCore".

Для этого было реализовано:

  1. Настроен процесс заведения набора в PimCore
  2. Предоставлены все необходимые данные для передачи данных о наборе в Navision
  3. В системе PimCore создана отдельная папка для хранени наборов "Наборы"

4.6.2.4 Настройка поля НДС

Изменено существующее поле в PimCore "НДС" на поле с выпадающим списком. 

Поле в PimCore выбирается автоматически на основании Выбранного ранее Типа товара. Есть возможность последующего его редактирования на самом товаре непосредственно

5 Техническое описание задания и технические требования (инфраструктура)

PIMCore размещается в инфраструктуре заказчика.

База данных инкрементально бекапируется во внешнее хранилище (S3 compatible) не зависимое от основной инфраструктуры

Ассеты загружаемые в PimCore так же бекапируются за пределы инфраструктуры во внешнее хранилище.

Системы потребители данных из PimCore должны использовать раздельные ключи доступа. Ни одна система не должна делить ключи доступа с иными.
