Created by Yaroslav Rudenok, last modified on Jun 24, 2023

Бизнес правила

Есть некоторые справочники, которые зависят от других справочников. В данном случае необходимо следовать последовательности заведения новых значений, что описаны ниже

Есть справочники, которые не зависят от других справочников. Значения в них можно просто добавлять при появлении новых

Есть атрибуты товара, что являются выбором из списка. Добавления новых значений для выбора в такие атрибуты может осуществить администратор системы либо через Support

Работа со справочниками для Framework

Если мы хотим расширить выпадающие списки для полей товара (номер тона, название тона и код HEX) либо завести новый Framework, то последовательностей добавления новых значений в связанные справочники следующая:

  1. Добавить новое значение в справочник "Настройка структуры вариантов"
  2. При необходимости: Добавить новое значение в справочник "Настройка структуры базовых значений"
  3. При необходимости: Добавить новое значение в справочник "Номер тона"
  4. При необходимости: Добавить новое значение в справочник "Название тона"
  5. При необходимости: Добавить новое значение в справочник "Код Hex"
  6. Добавить новое значение в справочник "Вариант структура измерения"
  7. Добавить новое значение в справочник "Вариант Структура Базовые Значения"

Справочник "Barcode"

При заведения нового товара и заполнении на нем поля Штрих-код, новое значение в справочник Штрих-код добавиться автоматически.

Та же ситуация произойдет, если штрих-код на товаре будет заполнен автоматически

Справочник "Категория ТМ дополнительна"

Значения в данный справочник можно добавлять не зависимо от других. Но данное значение на товаре заполняется из Справочника "Торговая марка", поле "Категория ТМ дополнительная"

Справочник "Набор полей и правил"

Данный справочник используется для формирования выпадающих списков для следующих атрибутов товара:

  • Семейство товаров
  • Тип товара
  • Пол

А так же для автоматического заполнения следующих полей товара:

  • Область применения (для фильтра в им) 
  • Область применения (для Тетры)
  • Категории всех уровней

При добавлении нового значения в справочник "Набор полей и правил" все его поля должны быть заведены ранее в соответсвующие справочники
