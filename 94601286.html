Мета

  • Розглянути особливості роботи з Hexagone як з основним постачальник Brocard в частині товарної номенклатури 

Головні тезиси

  • Глобальних відмінностей в роботі з Hexagone як з постачальником стосовно товарної номенклатури немає
  • Hexagone має мати можливість надавати основну інформацію по товару по спрощеній формі, тобто не одразу всі атрибути. Це враховано в прийнятому раніше рішенні про спрощений формат для всіх постачальників (первинний набір атрибутів)
  • Hexagone має стикерувати товар до моменту поставки товару на склад Brocard, для цього потрібно врахувати, які саме атрибути мають бути обов'язково заповнені для того, щоб цей процесс можливо було здійснити. Попередньо, всі поля, які необхідні для стикеру, є в PimCore, окрім поля "Состав" - в PimCore це поле називається "Состав товара (INCI)". Уточнити, чи інформація з "Состав товара (INCI)" підходить для стикеру.
  • Система ERP для Hexagone це Тетра Office Hexagone
  • Інформацію по товару для заведення заявки Hexagone бере з рахунків-фактур і переносить її в стандартну заявку-шаблон Excel, для заведення в PimCore підтягую файл в форматі .csv
  • Інформація по товару заноситься в ЕТР через завантаження Excel файлу, після цього шляхом процесу полуавтоматичного вивантаження інформація з ЕТР потрапляє в Тетра Office Hexagone

Відкриті питання

  • Проговорити процес стікерування і яка саме інформація потрібна для цього, яка зараз надходить в систему Labels і звідки вона туди потрапляє
  • Хто заповнює поле "Способ использования" в Labels
  • Чи підходить інформація з поля "Состав товара (INCI)" в PimCore для стікеру (для поля "Состав")? - Natalia Kolisnyk, Анастасія Нагорна
