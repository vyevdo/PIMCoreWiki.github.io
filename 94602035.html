Brocard : Ролі в PimCore

Created by Olga Tarasenko on Aug 05, 2022
#РольВизначенняФункції
1adminАдміністратор

Адміністратор системи повинен мати можливість:

  • Авторизуватись у системі
  • Змінити пароль
  • Створити обліковий запис
    • Постачальника
    • Бренд менеджера
    • Товарознавця
    • Адміністратора
  • Працювати з усіма брендами
  • Використовувати перспективу за замовченням для даних про продукт
  • Управляти всіма класами, у яких будуть зберігатися значення параметрів
  • Вносити дані про товар через інтерфейс Pimcore
  • Завантажувати дані про товар у форматі .csv
  • Міняти статус товару (В роботі, Підтверджено, Опублікований, В архіві)
  • Бачити статус товару (В роботі, підтверджено, опублікований, в архіві)
  • Мати можливість копіювати товар (всі атрибути крім штрих кодів та без копіювання варіантів товару) та/або варіанти товару (всі атрибути крім штрих кодів)
  • Мати можливість змінювати штрих коди для товарів в заархівованому стані.
  • Мати можливість проводити Bulk Change (масова зміна в множині карток товару) полів товарів
  • Управляти Custom Layouts
    • Створювати
    • Вносити масові зміни
    • Застосовувати для типів товарів
  • Мати доступ до меню (Actions) "доступні операції по товару"
    • Якщо товар в роботі (не підтверджений Бренд Менеджером або товарознавцем)
      • Зберегти товар
      • Редагувати товари
      • Підтвердити товар
    • Якщо товар підтверджений Бренд Менеджером або товарознавцем
      • Скасувати підтвердження товару
      • Опублікувати товар
      • Прийняти запит на зміну товару
    • Якщо товар опублікований Бренд Менеджером або товарознавцем
      • Мати можливість вивантажити дані з Pimcore у форматах .xls, .xlsx, .csv, .xml
      • Скасувати публікацію
      • Прийняти запит на зміну товару
      • Заархівувати товар
    • Якщо товар заархівований товарознавцем
      • Розархівувати товар

  • Авторизуватись у системі
  • Змінити пароль
  • Працювати з усіма брендами
  • Використовувати перспективу за замовчуванням для даних про продукт

  • Мати можливість проводити Bulk Change (масова зміна в множині карток товару) полів товарів.
  • Бачити статус товару (В роботі, підтверджено, опублікований, в архіві)
  • Мати можливість копіювати товар (всі атрибути крім штрих кодів та без копіювання варіантів товару) та/або варіанти товару (всі атрибути крім штрих кодів)

      • Підтвердити товар
    • Якщо товар підтверджений Бренд Менеджером або товарознавцем
      • Виконати операцію - створити варіанти (об'єднання товарів які відрізняються штрихкодом або одним атрибутом групуються в основний товар і його варіанти)
      • Скасувати підтвердження товару
      • Опублікувати товар
      • Прийняти запит на зміну товару
    • Якщо товар опублікований Бренд Менеджером або товарознавцем
      • Мати можливість вивантажити дані з Pimcore у форматах .xls, .xlsx, .csv, .xml
      • Скасувати публікацію
      • Прийняти запит на зміну товару
      • Заархівувати товар (необоротна для товарознавця операція)

  • Мати можливість проводити Bulk Change (масова зміна в множині карток товару) полів товарів.
  • Завантажувати дані про товар за допомогою .csv
  • Міняти статус товару (В роботі, підтверджено, опублікований)
  • Бачити статус товару (В роботі, підтверджено, опублікований, в архіві)
  • Мати можливість копіювати товар (всі атрибути крім штрих кодів та без копіювання варіантів товару) та/або варіанти товару (всі атрибути крім штрих кодів).
  • Мати доступ до меню (Actions) "доступні операції по товару"
    • Якщо товар в роботі (не підтверджений Бренд Менеджером або товарознавця)
      • Зберегти товар
      • Редагувати товари
      • Підтвердити товар

      • Скасувати підтвердження товару
      • Опублікувати товар
      • Прийняти запит на зміну товару
    • Якщо товар опублікований Бренд Менеджером або товарознавцем
      • Мати можливість вивантажити дані з Pimcore у форматах .xls, .xlsx, .csv, .xml
      • Прийняти запит на зміну товару

  • Використовувати кастомну перспективу для даних про продукт з коренем Products з Data Objects, при цьому медіа файли по продукту повинні зберігатися в Products з Assets. Таким чином фото не повинні розташовуватися в картці товару, а повинен бути встановлений зв'язок з картки товару до файлів, що зберігаються в Assets .
  • Вносити дані про товар через інтерфейс Pimcore
  • Завантажувати дані про товар за допомогою .csv
  • Завантажувати дані про товар за допомогою API
  • Бачити статус товару (В роботі, підтверджено, опублікований, в архіві)
  • Мати можливість копіювати товар (всі атрибути крім штрих кодів та без копіювання варіантів товару) та/або варіанти товару (всі атрибути крім штрих кодів).

      • Зберегти товар

      • Подати запит на зміну товару
