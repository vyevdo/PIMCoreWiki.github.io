Створення користувача

  1. Увійдіть в адмін панель Pimcore
  2. Перейдіть до Settings > Users & Roles > Users
  3. Натисніть ПКМ на папці "All Users" та виберіть "Add user"
  4. Введіть логін для нового користувача та натисніть "ОК"


Редагування користувача

  1. Увійдіть в адмін панель Pimcore
  2. Перейдіть до Settings > Users & Roles > Users
  3. Оберіть потрібного користувача, натиснувши на нього
  4. Відредагуйте інформацію для користувача та натисніть "Save"


Довідка для полей профілю користувача

Профіль користувача має 4 вкладки:

  • Settings - загальні налаштування
  • Workspaces - робочі простори, налаштування гнучкого доступу до об'єктів, фото і т.п.
  • The user is referenced in the following objects - інформаційне вікно, яке показує об'єкти, в яких є посилання на користувача (на даний момент такі зв'язки в проекті відсутні)
  • Key Bindings - налаштування гарячих клавіш для користувача


Вкладка "Settings"

Вкладка Settings має 6 розділів:

  • General
  • Admin
  • Permissions
  • Allowed types to create (Defaults to all)
  • Editor Settings
  • Shared Translation Settings

Розділ "General"

  • ID - ідентифікатор користувача, незмінний
  • Last Login - час останнього входу в адмін панель
  • Active - чи користувач активний (чи можливо виконати вхід в адмін панель)
  • Username - логін
  • Password - пароль, знак блискавки для автогенерації паролю
  • Two Factor Authentication - двофакторна автентифікація
  • Image - аватар
  • Firstname - ім'я
  • Lastname - прізвище

  • Email - електронна пошта, кнопка "Send Invitation Link" надсилає лист з посиланням для входу
  • Language - мова інтерфейсу для користувача
  • Roles - доступні ролі для користувача, можна обрати декілька за допомогою Ctrl та/або Shift
  • Perspectives - доступні перспективи для користувача, можна обрати декілька за допомогою Ctrl та/або Shift
  • Show welcome screen on startup - чи відображати привітання при вході
  • Memorize open tabs - чи поновлювати відкриті вкладки при наступному вході
  • Disable unsaved content warning - чи вимкнути попередження про те, що зміни не збережені
  • Show close warning - чи відображати попередження про закриття


Розділ "Admin"

  • Admin - зробити користувача суперадміном, в такому випадку ігноруються всі налаштування дозволів і користувач автоматично отримує всі дозволи та доступи
  • API Key - ключ API для виконання програмних запитів, значок блискавки автогенерує ключ
  • Login as this user in a different browser - отримати посилання для входу від імені цього користувача (використовується в цілях тестування)


Розділ "Permissions"

Містить дозволи для користувача, повний список та опис на сторінці Користувачі та ролі.

Примітка 1: окрім дозволів, отриманих тут, користувач також отримує дозволи ролей, які йому присвоєні.

Примітка 2: Дозволи "brand_*" призначені для контролю доступу бренд-менеджерів до товарів певних торгових марок. Створити дозвіл для нового бренду можна за допомогою кнопки "+ Brand", яка знаходиться над списком усіх користувачів.


Розділ "Allowed types to create (Defaults to all)"

  • Document Types - дозволені типи документів для створення (в проекті відсутні)
  • Classes - дозволені класи для створення (наприклад, товар, тип товару, вік тощо). Можна обрати декілька за допомогою Ctrl та/або Shift

Розділ "Shared Translation Settings"

Налаштування доступу до спільних перекладів (в проекті не використовується).


Вкладка "Workspaces"

Вкладка  "Workspaces" використовується для налаштування доступу до окремих елементів або папок та регулювання доступних дій для кожного правила. Для довідки див. Користувачі та ролі, розділ "Елементи дозволу - робочі простори".


Вкладка "Key Bindings"

Вкладка "Key Bindings" використовується для налаштування гарячих клавіш для конкретного користувача. Для зміни комбінації потрібно натиснути на потрібне поле і затиснути клавіші нової комбінації.
