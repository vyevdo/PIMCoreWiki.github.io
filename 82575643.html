    Валидация данных (Список проверок)

    As isTo beВозможные рискиПриоритет
    • Список проверок содержится в Excel файле
    • Список проверок неполный(например поля бренд, объем)
    • Файл с новыми товарами нужно конвертировать в *csv формат вручную
    • Часть проверок происходит в PIM: в случае расхождения данное поле подгрузится пустым либо не загрузится вся строка с товаром
    • Предоставить детальную инструкцию по созданию товаров в PIM Core, используя *csv формат
    • Добавить проверки из Excel файла в PIM
    • Добавить новые проверки в PIM с учетом поддержки актуальности справочников
    • Часть товаров заводится некорректно в PIM и передается в другие системы
    • Ручное исправление данных о товаре
    		Высокий

    Процесс создания набора

    As isTo beВозможные рискиПриоритет
    1. Завести набор в ETR как обычный товар ("тип товара" = Набор)
    2. Через автоматический импорт набор попадает в Tetra office
    3. Сделать импорт набора как единого товара в Navision
    4. Создать составляющие набора в Navision
    1. Добавить тип товара = "набор" в PIM Core
    2. Настроить привязку составляющих набора к товару "Набор" в PIM Core
    3. Список атрибутов, составляющих набор, должен совпадать со списком в Navision:
      1. Номер
      2. описание
      3. код варианта
      4. количество
    4. Выгружать данные о наборе с PIM Core в систему Navision

    Работа с вариантами

    As isTo beВозможные рискиПриоритет
    1. Объединить кодом Navision (Vendor Item No.) товары, которые нужно добавить в единую карточку товара в Excel файл
    2. Загрузить данный файл в Navision
    3. Система Navision автоматически проставляет вариант 
    4. В случае ошибок с объединением процесс их исправления достаточно трудозатратный
    1. Добавить поле код Navision в PIM Core
    2. Настроить правила объединения в варианты
    3. Настроить экспорт данных о вариантах PIM Core↔ Navision
    4. Возможно нужна дополнительная роль товароведа

    Работа с гигиеническим заключением

    As isTo beВозможные рискиПриоритет

    Данные для подачи документов получаем от поставщика. Дату и номер заключения храним в файлах. Единая система гигиенических документов (сканов) работает не совсем корректно

    Сертификация практически не проводится, но процесс аналогичный процессу получения гигиенического заключения

    • Настроить передачу атрибутов для получения гигиенического заключения в отдел легализации
    • Добавить в PIM Core атрибут Адрес производителя/поставщика
    • Настроить ввод данных в существующие колонки PIM Core
    • Настроить загрузку документов от поставщика, необходимых для подачи товара на гигиену (разного рода сканы сертификатов)
    • Создать единое место хранения гигиенических заключений 
    • Настроить сообщения о скором завершении того или иного гигиенического заключения

    Процессы бухгалтерии

    As isTo beВозможные рискиПриоритет

    Бухгалтерия работает в 1С УТП и Тетра Office

    Информации по товару в 1С недостаточно, приходится часть загружать в 1с из Тетры, часть остается в выгрузках Excel

    Часть атрибутов загружаются в 1С через импорт Excel

    Часть атрибутов не загружаются в 1С, но нужны в работе

    • Настроить трансфер из PIMCore в 1С УТП данных о товаре
    • Настроить создание некоторых товаров в PIM Core с дальнейшей их передачей в 1С УТП
    • Сделать выпадающий список в поле НДС в PIM Core редактируемым, дополнить существующими ставками (например, 7%)
    		Часть товаров, что создаются нельзя отнести к продукции. Возможно она не нужна в PIM Core

    Процесс работы с price list

    As isTo beВозможные рискиПриоритет
    1. Сотрудник отдела ценообразования заполняет Price list (типов Price list несколько)
      Для каждой позиции задается цена и дата, с которой цена начинает действовать, информация затягивается по ШК или коду Тетры
    2. Сотрудник отдела ценообразования загружает Price list  в Tetra office в виде Excel file
    3. Данные автоматически переходят из Tetra office → Navision
    4. В Navision цены отображаются списком и разделяются по коду продажи
    1. Настроить загрузку цен в PIM Core. Для этого нужно учесть, что PIM Core не может поддерживать их историчность
    2. Новая цена продажи с Price list = 3 колонки в PIM Core (Наименование Price list, цена продажи, "Дата с" = Дата начала действия цены )
    3. Настроить выгрузку price list PIM Core→ Navision
    4. Создать ограничения доступа к ценам в случае необходимости

    PIM не поддерживает историчность

    Дополнительные атрибуты товара в разрезе категорийных менеджеров Brocard

    As isTo beВозможные рискиПриоритет
    • Категорийные менеджеры Brocard работают с системой Тетра Office
    • Получают фото контент от поставщика по почте и пересылают его дальше в Департамент маркетинга розничной сети для размещения
    • Выполняют контроль подачи информации по товарной номенклатуре поставщиком

    Добавить следующие атрибуты в PIMCore с возможностью их редактирования в будущем:

    1. Представленность позиции – сеть/ИМ
    2. Позиция входит в ассортиментную матрицу или нет
    3. Категории ТТ для позиции – A/ A,B / A,B,C / только корнеры и т.д.
    4. Статус товара – новинка/снятый/архивный/лимитированный
    5. Статус бренда – аптечный, профессиональный, органический и т.д. – с возможностью выбора 3 вариантов (Натуральный+Органический+Веганский)
    6. Статус позиции – натуральный/органика/веган – с возможность выбора 3 вариантов
    7. Статус ценового сегмента в разрезе брендов/в разрезе СКЮ – Premium/LUX/Middle/Mass/Super Mass
    8. Соответствие товар-тестер
    9. Аналоги товара
    10. Сопутствующие товары

    Дополнительные атрибуты, что загружает Е -com

    As isTo beВозможные рискиПриоритет

    1. Поставщик предоставляет данные о товаре и фото в контент отдел

    2. Контент отдел передает данные для загрузки в E - com

    3. Атрибуты загружаются через Excel файл в Navision

    4. Фото загружается массово до 100 штук за раз

    5. Главное фото идет на карточку товара (формат ШХ_a)
    6. Далее можно загрузить фото на варианты (формат ШХ_1)

    7. Свотчи и фото наборов загружаются немного по другой логике

    Часть полей в Navision нельзя править автоматически

    1. Добавить недостающие поля в PIM Core
    2. Добавить возможность загрузки фото в PIM Core на главный штрих код, свотчи, наборы
    3. Определить ответственного за наполнения атрибутов товара (поставщик, E-com)
    4. Настроить обмен данными PIM Core → Navision
    		Нехватка места в хранилище данных
