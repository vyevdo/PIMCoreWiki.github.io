Цель

  • Посмотреть программу для стикеровки на складе Брокард
  • Оценить, какие атрибуты в ней есть и откуда по ним поступает информация

Результаты встречи

  • На складе Брокард используется система Labels 2, программа реализована в виде Excel файла
  • Данные для программы хранятся в базе данных склада
  • В месяц есть необходимость печатать 20-30 тыс стикеров

Система Labels 2

В программе описаны три вида сущностей

  • Поставщик (пока у Брокарда только один поставщик)
  • Товары
  • Упаковочный лист

Поставщики

Cодержит следующую информацию

  • Код
  • Название
  • Статус
  • ШаблонТекста1 - используется на данный момент
  • ШаблонТекста2
  • Описание

Шаблон собирается из данных этого файла (см.скрин), где Good - это товары и соответствующие названия полей этой вкладки, Packlist - упаковочный лист, информация о конкретной поставке.

Список полей, которые необходимы для стикерования

  • Артикул - первые 4 символа артикула, который указан на товаре
  • Марка товара
  • Название оригинальное
  • Название на украинском языке
  • Объём
  • Состав (ингредиенты)
  • Страна
  • Дата производства
  • Срок хранения
  • Меры предосторожности
  • Адрес производителя

Товары

На вкладке Товары отображается вся необходимая информация по товару. Данные присылаются поставщиком в виде базы данных MS Access, сохраняются в базе данных Брокарда и далее используется для формирования текста ярлыка

Нажимая кнопку Получить отчет информация по товарам обновляется

Связь с товаром в Тетре можно найти только по артикулу (в том случае, если он заведен идентично правильно)

Список полей

  • Поля, которые используются для печати (добавочные поля) - на данный момент функционал не используется
    • GoodTemplate 
    • FontName
    • FontSize
    • PrintTemplate
  • Поля (данные), которые присылает поставщик в виде файла базы данных
    • GoodАртикул - первые 4 символа артикула, который указан на товаре
    • GoodГруппа товара - похожее на определение из Тетры, но все-таки не связанное, определяет поставщик
    • GoodМарка товара
    • GoodНазвание оригинальное
    • GoodНазвание русское - название на украинском языке
    • GoodОписание набора
    • GoodМеры предосторожности
    • GoodОбъём
    • GoodСостав (ингредиенты)
    • GoodПроцент Денатурата - не используется
    • GoodАдрес производителя
    • GoodСрок годности - не указывается на этой вкладке
    • GoodСерт орган - не используется
    • GoodСоздана
    • GoodИзменена
  • GoodНадпись на стикере - то, что сформировано для стикера

Упаковочный лист

Файл Excel Упаковочный лист высылает поставщик. Данные из данного файла импортируются в базу данных и оттуда уже загружаются в программу Labels 2. Файл в т.ч. содержит вес паллеты, вес коробки, размер, но эти данные не используются.

На вкладке Упаковочный лист отображается вся необходимая информация по поставке товара. Также в скобках указаны названия полей из предоставленного Олегом файла с выгрузкой.

  • Упаковочный лист (PackListID) - референс на Excel файл поставки
  • Паллета (PackListPallet) - ШК паллеты
  • Коробка (PackListCarton) - ШК коробки
  • Product (PackListProduct) - полный артикул
  • Articul (PackListArticul) - первые 4 значения артикула
  • Країна (PackListCoO) - код страны
  • ManufacturingCode (PackListManufacturingCode) - батч код, из которого берется информация по дате производства
  • Дата производства (PackListDateProd)
  • Годен до (PackListDateTo)
  • Quantity (PackListQuantity) - количество товаров
  • Напечатать  - расчетное поле с указанием количества страниц со стикерами, которые нужно распечатать
  • СтикерШаблонТекста1 - шаблон стикера
  • PackListProductDesc - описание товара
