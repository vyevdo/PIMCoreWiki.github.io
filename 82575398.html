Результаты встречи

  • На встрече подробно рассмотрели файл с правилами для поставщика и PIMCore (во вложении)
  • В зависимости от типа товара присваиваются атрибуты (есть обязательные и не обязательные)
  • В Excel файле, который рассматривался на встрече, цветами отмечены правила по ключевым атрибутам
    • Зелена заливка = обов’язковий атрибут. Без присвоєння постачальник не може закрити заявку
    • Жовта = атрибут може бути прив’язаний. Постачальник може вибрати значення
    • Червона = атрибут не може бути присвоєний цій позиції
  • Поставщик заполняет заявку в Excel согласно установленных правил для номенклатуры и потом импортирует ее в PIMCore. В случае, если обязательные поля не заполнены или заполнены неверно, PIMCore выдаст ошибку
  • В PIMCore есть уведомления/всплывающие окна о типе ошибки (пока только для грубых ошибок) на рус/укр языке, в будущем планируется сделать всплывающие подсказки и для других ошибок
  • В Excel форме учтено одно правило, а именно, при выборе Назначения в колонке Тип товара в выпадающем списке будет список соответствующий конкретному назначению
  • В Excel форме нет выпадающего списка по брендам (ТМ), т.к. этот список постоянно обновляется, поле текстовое. Это поле не импортируется в PIMCore
  • Новый бренд (ТМ) для начала дожен быть заведен в PIMCore товароведами, к нему привязывается следующие атрибуты, которые предоставляет поставщик:
    • класс ТМ
    • владелец ТМ
    • страна происхождения
  • После импорта данных поставщиком из формы товароведы обновляют в PIMCore информацию по ТМ
  • В PIMCore к типу товара привязано назначение, в форме Excel наоборот
  • Категорий в форме Excel нет. В PIMCore категория (несколько уровней через "/") будет присваиваться автоматически в зависимости от связки тип товара+назначение+пол+признак аптечного бренда. В зависимости от типа товара категорий может быть несколько
  • Область применения для Тетры привязана к типу товара. В PIMCore есть типы товаров, у которых привязана область применения для Тетры и проставляется автоматически, а есть те, где ее должен проставить поставщик и если он этого не сделает, PIMCore выдаст ошибку. Также есть ситуация, когда указанная поставщиком область применения не соответсвует той, которуя автоматом подтянул PIMCore, в таком случае нужно рассматривать этот кейс отдельно и уточнять, какую область применения отображать
  • Область применения для Navision проставляется автоматически в зависимости от типа товара. Можно учесть в форме Excel, чтоб область применения для Navision подтягивало формулой в зависимости от выбранной области применения Тетра
  • В PIMCore при импорте данные по товару заходят по ключу Название для PIMCore (Название товара на укр.языке + штрих-код ( через дефис ))

 

Правила от товароведов

Для заведення номенклатури в Pim Core послідовність має бути такою:

  1. Бренд
  2. Стать
  3. Призначення (наприклад, якщо це нішевий бренд, то можна вибрати тільки призначення Niche, якщо це дитячі товари (пол Д), випадає тільки Health&Beauty) - сделать во второй этап разработки правил в PIMCore
  4. Відповідно до призначення зі списку типів товарів цього призначення вибираємо потрібний тип товару
  5. Вибираємо для товарів область призначення (тільки для товарів, яким не присвоюється така область автоматично). Див.лист ОП если присваивает поставщик
  6. Відповідно до типу товара заповнюємо атрибути
  7. Автоматично присвоюються такі дані:
    1. Категорія/категорії всіх рівнів (за сукупністю даних в пунктах 1-5). Крім товарів, для яких потрібно додатково вибрати категорію всіх рівнів, такі типи товарів виділені жовтим
    2. Область применения Тетра через тип товару. Крім товарів, для яких це визначає постачальник (такі товари виділені жовтим)
    3. Область применения NAV через тип товару
    4. Лінія на укр.мові для існуючих ліній (англ версію вибирає зі списку постачальник)
    5. Країна похождення (для існуючих брендів)
    6. Виробник (для існуючих брендів)
    7. Ознака імпортного товару (для існуючих брендів)
    8. Назва товару для PimCore

Вопросы

  • проверить, достаточно ли привязки атрибутов к типу товара или нужно привязывать к связке тип товара + назначение
  • вынести отдельно в файле правила по присвоению категорий и атрибутов
  • получить новый шаблон-заявку для поставщика
