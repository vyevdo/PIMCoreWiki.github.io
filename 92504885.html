Brocard : Общие материалы

Created by Yaroslav Rudenok on Jul 01, 2022
  1. Коллеги, организовываю встречи по перечисленным системам.
    Предлагаю пройти по порядку по всем системам в указанные даты. Если будут накладки - перенесем.

  2. Все эти программы работают с одним источником данных – Тетра. Также все эти программы работают с одной базой данных (кроме Лейблс), в которую каждый день выгружаются товарный справочник, отгрузки, продажи, остатки. Остальная НСИ берется либо из Тетры, либо заводится и хранится непосредственно в соответствующей программе. Соответственно, при выводе Тетры из эксплуатации если ничего не предпринимать, то программы перестанут работать.
  3. Для сохранения функциональности этих программ будет 2 этапа:
    1. На первом этапе будет переключен источник данных для этих программ с Тетра на Хранилище или на NAV. Как только необходимые данные для работы программ станут доступны в Хранилище, будет проведено детальное планирование переключения источника с привлечением ключевых пользователей к проверке результатов.
    2. На втором этапе (после запуска NAV в ЦО) будет проведен повторный анализ с целью реализации функциональности эти программ непосредственно в NAV или других системах (например, Стикерс может быть реализован в PIMCore)
  4. Отдельное внимание необходимо уделить существующим регулярным отчетам, которые строятся из Тетры и рассылаются по электронной почте. По мере накопления данных в Хранилище отчеты будут переделываться на работу с Хранилищем.
