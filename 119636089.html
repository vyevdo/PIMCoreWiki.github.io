Ціль

Налаштувати швидку взаємодію Постачальник <=> Брокард для вирішення питань, що виникають  у постачальника


Схема


BPMN diagram: Chatwoot

Схема без чата

BPMN diagram: Test_Chatwoot


Відкриті питання

  1. Як завершувати звернення?
  2. Чи треба десь реєструвати відповіді?
  3. Чи можливий діалог з постачальником під час вирішення тікету?
  4. Чи треба Якась статистика по результату?
  5. Чи може постачальник перевідкрити тікет?
  6. Чи треба якесь тестування, прийняття робіт, додаткове погодження?
  7. Яку логіку плануємо закласти для формулювання питання та його спрямовлення до особи, що буде його вирішувати?
  8. Чи треба нам облікова система для ведення осіб, що задіяні у підтримці?
  9. Налаштування інтеграції Chanwoot ↔ ITSM
  10. Тригер для автоматичного переходу з Dialog Flow у Chatwoot

Бізнес правила:

  1. Тільки товарознавець спілкується з постачальником
  2. Товарознавець змінює відповідь від підтримки перед тим як відправити результат постачальнику
  3. Можна передати чат з Chatwoot в DF назад змінивши статус події на pending.
