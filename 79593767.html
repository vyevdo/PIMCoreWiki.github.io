Цель

  • Оценить, какими данными по товару пользуются бренд-менеджеры Hexagone

Результаты встречи

  • Hexagone является поставщиком как Брокарда, так и других сетей
  • Специалисты заполняют информацию по размерам, палетам и другим "складским" параметрам в случае ее предоставления поставщиком, поля не обязательные. Если этой информации нет, то за ней обращаются на склад, где ее берут из WMS и предоставляют по запросу
  • Данных в Тетра для Брокарда достаточно, для других сетей иногда приходится обращаться за информацией на склад. Данные, запрошенные на складе дополнительно нигде исторически не хранятся.
  • Информация по товару и изменениям по нему передается через товароведов в формате Excel через шаблон и точечно по изменениям, правки в систему вносят товароведы, Hexagone править в PIMCore сам не может
  • Для стикеровки товаров существует программа Labels. Она подтягивает часть информации из Тетры автоматически
  • Данные для заявки для гигиенического заключения и сертификации берутся из Тетры, остальную информацию, а именно, название фабрики и ее адрес, предоставляет поставщик. Эти дополнительные параметры вносятся потом только в программу для стикеров
  • Информация после получения гигиенического заключения и сертификата вносится в Тетру отделом легализации
  • В Тетру загружается файл со списком товаров для того, чтоб в программе Orders (которой пользуются бренд-менеджеры) данные подсвечивались жирным и было видно топ продаж. Неактуальный товар в Orders светится красным (Тетра отображает самостоятельно по статусу, который подают бренд-менеджеры товароведам для загрузки информации в Тетра).
  • Есть показатель Новинка, который не работает в системе Тетра. Должен отображаться как новинка, если с момента прихода товара еще не прошло 6 месяцев.
  • Информация по сроку товара ведется складом в системе WMS и этой информацией пользуются бренд-менеджеры (выгруженным отчетом, который размещают на диске общего доступа).
