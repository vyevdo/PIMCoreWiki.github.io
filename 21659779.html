Brocard : Настройка доступных атрибутов и их значений

Created by Valeriy Kulyk on Aug 28, 2019

Предложение: реализация через Сustom Layouts.

Пример: 

Возьмем существующую структуру класса Product - 8 вкладок.

Список значений поля “product_type” - “Тип товара” (в примере будут использованы только значения “activator” и “base”)

Список значений поля “purpose_main” - “Назначение (основное)”

Предположим, что у нас есть 2 задачи:

  1. Для товаров типа “activator” убрать вкладку “Content” и убрать поле “purpose_additional” из вкладки “Characteristics”

  2. Для товаров типа “base” в поле “purpose_main” оставить только варианты “MakeUp” и “Untype”


Задача 1

  1. В настройках класса “products” создаем Custom Layout для типа товара “activator”, с помощью описания (или, возможно, другого поля) указываем название типа товара

  2. С окошка “Сlass” перетягиваем весь Layout, и потом через ПКМ - Delete удаляем ненужные нам вкладки и поля


Задача 2

  1. В настройках класса “products” создаем Custom Layout для типа товара “activator”, с помощью описания (или, возможно, другого поля) указываем название типа товара

  2. С окошка “Сlass” перетягиваем весь Layout, и потом заходим в настройки поля “Назначение (основное)”, где в разделе Selection Options удаляем ненужные нам варианты



Вопрос 3 - Настройка выбора доступных объектов для отношений 1 к 1, 1 к многим и тп

Предложение: настройка через атрибуты объекта.

Пример: Допустим, что для выбора цвета в объектах класса “Product” мы создали класс “Сolor”. Нам нужно указать, что цвет “Морская волна #009688” доступен только для типов товара “activator” и “base”.

  1. Создаем класс “Сolor” с атрибутом множественного выбора (мультиселект) “Доступные типы товара”, набор значений дублируем из типов товара в классе “Product”

Создаем цвет “Морская волна #009688” и указываем возможные варианты типов товара в поле “Доступные типы товара”
