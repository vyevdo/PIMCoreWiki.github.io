Brocard : Идентификатор родителя

Created by Yaroslav Rudenok, last modified on Jun 09, 2023

Словарь

НазваниеОписание
Товар - родительТовар у которого в системе существует товар, в котором поле "Идентификатор родителя" заполнено данным объектом (товаром)
Дочерний товарТовар, у которого поле "Идентификатор родителя" заполнено


Функциональные требования

При ручном вводе:

  1. Создание товара
    1. Автозаполнение атрибутов нового товара согласно данных  о его родители
      При создании нового товара в случае, если поле "Идентификатор родителя" заполнено , то система на новом товаре автоматически должна заполнить некоторые атрибуты, скопировав их с товара, что указан в поле "Идентификатор родителя". Заполнение происходит после нажатия одной из кнопок на товаре "Save" (Для неопубликованых) или "Save & Publish" (Для опубликованных)
      Перечень атрибутов указан в файле Копия Поля Нав_PimCore, вкладка "Атрибути товара финал" колонка "Копіювати атрибут з батьківського товару при створені?". Условие: значение ячейки = "Да"
      Если какое -то поле на товаре было заполнено ранее, то приоритетным значением является значение, что указано на родителе. То есть поля перезатрутся
    2. Если при создании нового товара поле "Идентификатор родителя" не заполнено, то никаких действий по автозаполнению полей с товара родителя делать не нужно
  2. Редактирования данных на товаре 
    1. Если на товаре -родители меняется значение атрибута ,что указано в файле Копия Поля Нав_PimCore, вкладка "Атрибути товара финал" колонка "Копіювати атрибут з батьківського товару при редагуванні?", Условие: значение ячейки = "Да", то на всех дочерних товарах даного товара  - родителя (опубликованные и неопубликованные) значение этого атрибута так  же должно поменяться
    2. На дочернем товаре необходимо запретить изменять атрибуты, что удовлетворяют условию значение ячейки = "Да" в файле Копия Поля Нав_PimCore, вкладка "Атрибути товара финал" колонка "Копіювати атрибут з батьківського товару при редагуванні?". Если пользователь пытается изменить значение данного атрибута, то при попытке сохранить изменения должна отображаться ошибка "Изменения даного атрибута не возможно. Необходимо изменить его на товаре-родителе" & указать штрихкод товара - родителя
    3. Если поле "Идентификатор родителя" на товаре изменяется, а именно либо ставится другой объект, либо заполняется впервые, то перечень атрибутов заново копируется с указанного товара - родителя
    4. Если поле "Идентификатор родителя" на товаре изменяется, а именно делается пустым, то никаких изменений не делаем

При импорте

  1. Создание товара
    1. Автозаполнение атрибутов нового товара согласно данных  о его родители
      При создании нового товара в случае, если поле "Идентификатор родителя" заполнено , то система на новом товаре автоматически должна заполнить некоторые атрибуты, скопировав их с товара, что указан в поле "Идентификатор родителя". Заполнение происходит при импорте
      Перечень атрибутов указан в файле Копия Поля Нав_PimCore, вкладка "Атрибути товара финал" колонка "Копіювати атрибут з батьківського товару при створені?". Условие: значение ячейки = "Да"
      Если какое -то поле на товаре было заполнено ранее, то приоритетным значением является значение, что указано на родителе. То есть поля перезатрутся
    2. Если при создании нового товара поле "Идентификатор родителя" не заполнено, то никаких действий по автозаполнению полей с товара родителя делать не нужно
    3. Если при создании нового товара поле "Идентификатор родителя" заполнено некорректно ( то есть аткого товара родителя в системе нет), то выдаем критическую ошибку "You filled the wrong product - parent_шв. Please refill the Parent ID and re-upload the file"
  2. Редактирования данных на товаре
    1. Если на товаре -родители меняется значение атрибута при загрузке файла,что указано в файле Копия Поля Нав_PimCore, вкладка "Атрибути товара финал" колонка "Копіювати атрибут з батьківського товару при редагуванні?", Условие: значение ячейки = "Да", то на всех дочерних товарах даного товара - родителя  (опубликованные и неопубликованные) значение этого атрибута так  же должно поменяться
    2. На дочернем товаре при импорте необходимо запретить изменять атрибуты, что удовлетворяют условию значение ячейки = "Да" в файле Копия Поля Нав_PimCore, вкладка "Атрибути товара финал" колонка "Копіювати атрибут з батьківського товару при редагуванні?". Если в файле импорта значения данного атрибута отличаются от товара - родителя, то при попытке загрузки файла должна отображаться некритическая ошибка "Изменения даного атрибута не возможно. Необходимо изменить его на товаре-родителе" & указать штрихкод товара - родителя. При этом файл загружаем, но значения атрибутов не изменяем
    3. Если при импорте товаров, что уже существуют в системе поле "Идентификатор родителя" на товаре изменяется, а именно либо ставится другой объект, либо заполняется впервые, то перечень атрибутов заново копируется с указанного товара - родителя
    4. Если при импорте товаров, что уже существуют в системе поле "Идентификатор родителя" на товаре изменяется, а именно делается пустым, то никаких изменений не делаем
