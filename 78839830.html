Brocard : 2021-12-20 - Движение товара

Created by Olga Tarasenko, last modified on Jun 15, 2022

Движение товара в системах:

  1. Стандартная форма в виде файла Excel, которая отправляется поставщику для заполнения (с ограничениями и правилами заполнения). Содержит все колонки, которые необходимы для внесения данных в ЕТР (данные для ТЕТРА дублируют данные ЕТР, в ТЕТРА формируется код ТЕТРА для ЕТР и Navision).
  2. Поставщик самостоятельно затягивает данные (эту форму Excel) в PIMCore 
  3. Товаровед экспортирует данные из PIMCore и загружает в ЕТР
  4. Из ЕТР данные автоматически загружаются в ТЕТРА
  5. Из ТЕТРА выгружается файл Excel в стандартном формате, содержит код ТЕТРА
  6. Через ВПР подтягивается код ТЕТРА и таблица трансформируется для дальнейшего импорта в Navision с колонками в определенном порядке (готовится вручную). Коды тетра для Navision могут объединять несколько уникальных кодов ТЕТРА для вариантов, которые будут отображаться в карточке товара в Navision. Список соответствий ведется вручную.
  7. Данные импортируются в Navision по определенному шаблону через файл Excel с набором определенных атрибутов
  8. Из PIMCore выгружается файл Excel с данными по товару, который загрузил туда поставщик. Файл уже содержит код ТЕТРА, который в PIMCore проставляется автоматически
  9. После этого по нему специалист импортирует в Navision другой набор атрибутов по штрихкоду. Этот набор атрибутов загружается только в эту систему и является фильтрами для интернет-магазина
  10. Цены проставляются в ТЕТРА через импорт Excel файла по штрихкоду, называются “прайс листы”, цены автоматически загружаются в Navision, где они называются “ценовые группы”. В PIMCore цены тоже есть.


Если происходит изменение названия в карточке товара, то штрихкод остается неизменным, но все магазины информируются об изменении и ценники перепечатываются. 

Информация по новому поставщику/бренду заводится вручную в каждую систему. Сначала она вносится в ЕТР, потом в Navision.


Наталья: Нижче в таблиці прописані основні можливості, які є в ЄТР, та вказала як це відбувається в NAV та PimCore. По PimCore не впевнена у трьох пунктах, поставила знаки питання.

До додаткових функцій/ задач, які ми обговорювали, додам:

  1. Створення папок для фотографій по постачальникам з обмеженим доступом. В PimCore фото повинні заливатися по певним правилам: Папка постачальника/папка ТМ/папка з датою подачі. Папка з фото одного постачальника, повинна бути невидима для іншого постачальника.
  2. Можливість дробленого експорту по товарам та атрибутам.

1імпорт файлу в довільному шаблонітаквикористання шаблонувикористання шаблону
2імпорт інформації одним файломтакімпорт проводиться декількома файлами залежно від к-сті атрибутівтак
3детальний протокол фіксування любих змінтакні?
4пошук по неактуальному ШК (тобто в базі даного ШК не має, але зафіксовано в протоколі)такні?
5простий інтерфейс (вся важлива іфнормація знаходиться на виду, не потрібно переходити на нові вкладки)такніні
6можливість робити правки по групі товару (наприклад 10 ШК присвоїти одну лінію)такні?


Открытые вопросы:

  1. Файлы Excel импорта и экспорта в/из систем (от Натальи), по каким полям происходит поиск соответствия/происходит подтягивание, а также правила внесения информации по товару в формате 0/1.
  2. Уникальные ли штрихкоды (может быть ли несколько товаров под одним штрихкодом)?
  3. Может ли быть товар под одним штрихкодом с разными названиями?
