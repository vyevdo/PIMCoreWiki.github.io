Атрибуты

Варианты

  • В Тетре каждый ШК это отдельный вариант, соответственно он имеет свой индивидуальный код Тетры
  • В Navision товар объединяется в одну карточку товара
  • Варианты можно импортировать через Excel файл, а можно добавлять вручную
  • Бывают случаи, когда товар не изменился, но поставщик изменил ШК. В таком случае его все равно нельзя добавить как вариант, Navision это не пропустит, нужно создавать новую карточку товара
  • Для разных назначений товара разные правила объединения. Ниже перечислены те, в которых они могут быть:
    • Fragrance
    • MakeUp
    • Untype - аксессуары
    • Face skincare
    • Body skincare
    • Hair care
    • Men's care
    • Health and beauty
    • Niche - тоже относится к парфюмерии

Логика объединения вариантов

  • Должны совпадать:
    • ТМ
    • Линия
    • Тип товара
    • Эффект
  • Для парфюмерии и ухода (Fragrance, Niche, care) - разный объем
  • Для MakeUp - одинаковый объем, разные оттенки
  • могут быть исключения в виду разнообразия продуктов

Вкладка Варианты в карточке товара

Пример содержит два ШК

Атрибуты вариантов:

  • Barcode No.
  • Variant Code
  • Valid
  • Description
  • ОБЪЕМ_ПАРФУМ
  • Dimension 1 Weight
  • Variant Weight
  • Item Status Code
  • Replen. Item Store
  • Replen. Sales History Adj..
  • Replen. from Warehouse
  • Planned Cross Dock Qty.
  • Image Set

На другой вкладке показано, что каждому штрихкоду варианта соответствует код Тетра. Код варианта состоит из код товара Navision+код варианта (в примере это 000002115/00000232). Код варианта система генерит сама

Импорт общих атрибутов происходит по коду товара в Navision. Для вариантов привязка идет по ШК и коду Тетра


Удаление вариантов

(Удаление вариантов обсудили на встрече 03.02.2022, с 50 минуты)

  • Бывает необходимость вывести вариант из карточки товара в отдельный товар или изменить объем/цвет варианта. В таком случае ШК не меняется, т.к. товар один и тот же и Navision товар с одинаковым ШК не пропустит
  • В карточке товара ничего не удаляется, чтоб сохранить всю историю, но убирается ШК у варианта, вариант скрывается на сайте, закрываются по нему продажи, добавляется новый вариант с тем же ШК и правильно информацией (если речь об объеме или цвете)
  • Для вынесения варианта в отдельную карточку действия аналогичные, только вместо нового варианта добавляется новый товар в Navision
  • Как правило, решение принимается по объему продаж, т.е. скрывается та позиция, которая восстребована чуть меньше, чтоб не убирать активно продаваемые позиции с сайта
  • Все вышеописанное связано с правилами работы в Navision и на текущий момент не может работать по другому

Справочники

В Navision ведутся отдельные справочники (структура базовых значений по ТМ, номер тона, цвет тона, объем и т.д.).

В случае, если информация по цветам будет заводится в PIMCore, и будет какое-то новое значение, сначала нужно будет обновить справочники в Navision, чтоб потом иметь возможность загрузить товар из PIMCore в Navision

Наборы

  • Наборы никогда не объединяются в варианты
  • У наборов всегда тип товара Набор
  • Наборы пока в PIMCore не заводились, но будут заводиться попозиционно. На текущий момент наборы заводятся в ЕТР попозиционно (как отдельный товар); там, как и в Тетре, составляющих набора нет, составляющие появляются в Navision
  • Набор может состоять как из товаров, так и из миниатюр, семплов
  • Если в набор входит позиция, которая доступна к продаже, то она будет отдельно заведена в Navision, и в этом случае код этого товара подтягивается в составляющие набора. Если это миниатюры, семплы, то по ним создаются "пустышки", чтоб их можно было добавить в набор, они имеют статус "ингредиент набора", не имеют ШК, код Тетры генерит Navision, имеют код Navision и название.
  • Можно указать количество штук в наборе для одинаковых позиций
  • Позиция Набор в Navision наполняется другими атрибутами аналогично обычным товарам
  • Обратно в Тетру дополненная информация по наборам не транслируется
  • Для импорта в Navision указывается следующая информация по наборам:
    • код товара Тетра
    • ШК
    • ТМ
    • Линия
    • назначение
    • тип товара
    • артикул
    • код УКТ ЗЕД
