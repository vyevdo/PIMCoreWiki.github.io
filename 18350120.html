Краткое описание

В Pimcore есть встроенная функция импорта, которая работает с файлами в формате CSV. Для импорта можно создавать свою конфигурацию (т.е привязку колонок файла к атрибутам объекта) для сохранения настроек при будущих импортах файлов с одинаковой структурой

Импорт

Для проведения импорта нужно проделать следующий набор шагов:

  1. Войти в админ панель: {+}https://brd.nodeart.app/admin+

Вкладка "Column Configuration"

Эта вкладка предназначена для настройки колонок в файле. Принцип работы с этой вкладкой такой же, как и с настройкой сетки товаров.
1 - Поле для поиска атрибутов объекта (товара)
2 - Атрибут объекта, который можно перетянуть в правую часть, чтобы привязать его к определенной колонке в файле
3 - Колонка файла, к которой привязан атрибут объекта
4 - Кнопка для сохранения настроек

Вкладка "Resolver settings"

Эта вкладка предназначена для определения того, как система будет искать, существует ли импортируемый товар или же он создается с нуля. Варианты определения: по ID, по полному пути к файлу, по названию товара, по значению какого-то атрибута, или же отдельный способ, описываемый с помощью кода. Описание всех вариантов есть на сайте Pimcore, поэтому приведу только объяснение работы самописного определения, которое соединяет категорию и название товара.


Skip head row - пропускать ли первую строку в файле (в первой строке обычно содержится название колонки)
Resolver strategy - это вариант определения. В нашем случае, код который отвечает за определение объекта, написанный нами, поэтому значения варианта - "Code"
Column - тут определяется колонка, которая служит основой для определения, в нашем случае тут должна быть колонка с категорией объекта
PHP Class - тут записано имя класса, который будет определять объект
Additional Data - сюда мы передаем для нашего класса номер колонки, в котором записано название объекта
Type - тип (объект, папка, вариант) того, что мы хотим создать, в нашем случае - объект
Create on demand - создавать ли объект, если его раньше не было
