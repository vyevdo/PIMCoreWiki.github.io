Результаты встречи

  • Загрузкой медиаконтента на сайт Брокард занимается департамент Еком
  • Видеоконтент на текущий момент не используется
  • Фотоконтент товаров для сайта Брокард загружается в Navision в связке с товаром и вариантами
  • Загружаются фото товаров и свотчи (оттенки) (небольшие изображения), свотчи привязываются к вариантам товаров
  • Все фото предоставляются департаменту поставщиком
  • Загрузка фото происходит в 3 этапа
    • необходимо выделить главное фото для товара, оно подвязывается к карточке товара, формат для заведения "ШХ_a"
    • для каждого варианта фото загружается отдельно, формат для заведения "ШХ_1"
    • для наборов заведение фото отличается, т.к. единица изменения не штука, а набор, формат для заведения "ШХ_a/b/c..", фото просто загружаются последовательно, главное фото не выделяется
    • свотчи загружаются через опцию "не привязывать к товару", позднее они дополнительно привязываются по коду в админке
  • Фото загружаются массово (до 100 фото за раз), но есть возможность в карточке заменить одну фотографию
  • Метчинг фото в системе происходит по ШХ
  • Фото и другая информация по товарам из Navision реплицируется на базу Еком для интернет магазина раз в сутки по регламенту (по необходимости через ИТ запрос это можно сделать вручную) и после этого информация из базы попадает через импорт каталога на сайт в полуавтоматическом режиме (девопс запускает процедуру импорта каталога раз в неделю)
  • Дополнительные атрибуты, которые присваивает департамент в Navision:
    • Признак публикации (publish estore) - признак отображения на сайте (отображается или скрывается информация после импорта)
    • Color
    • Tone number
  • Информация в систему заводится по мере поступления данных (фото, описание товара, как использовать, список атрибутов) от контент отдела, как правило раз в неделю
  • В контент отдел информация по товару приходит от поставщика, там ее обрабатывают, проверяют и вносят данные в принятый внутренний шаблон
  • Информация в Navision загружается в формате Excel с определенными названиями колонок и их очередностью, есть возможность править вручную в карточке товара
  • В зависимости от атрибута, изменения по товару можно сделать через импорт файла Excel, либо необходимо вручную сначала удалить атрибут, а потом загрузить новое значение

Еком:

Примеры файлов, с которыми идет работа во время загрузки контента в Navision. Примечание: все атрибуты привязываются по штрих-коду к каждому варианту. 

  1. Нооменклатор_каталог_27.12 — пример файла с описаниями и атрибутами, который я получаю от контент-отдела. Происходит в таком формате: от Кристины Кисловой я получаю ссылку на папку, где находится файл Нооменклатор_каталог_дата, основные фото для товаров и свотчи.
  2. Нооменклатор_каталог_ОБЩАЯ — предыдущий файл, в который я подтягиваю данные из Navision: код товара, код варианта, код тетры (последний не используется при загрузке контента, это данные для отдела ценообразования).
  3. Описания/Способ РУ/УКР — описания к товарам. Загружаю отдельно каждую языковую версию. Примечание: описания подвязываются по коду Navision к основной карточке товара. 
  4. Признак публикации — атрибут Publish in E-Store.
  5. Семейство цветов — атрибут Family colour.
  6. Номер тона — атрибут Tone number/Номер тона.
  7. Название цвета — Color/ Цвета тона.
  8. Файлы для привязки свотчей к товарам:
    1. Hex + THUMBNAIL — код HEX;
    2. Hex + Barcode — атрибут Thumbnail
