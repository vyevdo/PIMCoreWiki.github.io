Brocard : Работа товароведов по ведению ассортимента товаров

Created by Yaroslav Rudenok, last modified by Olga Tarasenko on Jan 27, 2022
  1. В PIM core поставщик загружает excel file с новыми позициями либо он может создать позицию в PIM core вручную на платформе PIM core. Список полей в файле "Поля по всім програмам.xlsx"(69 полей)
  2. В ETR нужно указать "Номенклатурный номер". В каждой марке есть номенклатурный номер. Товаровед смотрит какой последний номер, и заполняет это поле с новыми номерами
  3. В номенклатурном номере должно быть не более 10 символов(требований по формированию номеров нет). Как правило он формируется: Сокращенная марка & Порядковый номер
  4. Перечень номенклатурных номеров хранится только в ETR
  5. Номенклатурный номер добавляется в excel файл, что загружается в ETR
  6. Код тетры ETR присваивает автоматически при заведении товара
  7. Информация с ETR в Tetra приходит автоматически.
  8. В Tetra нужно добавить поле "Область применения". Она загружается через Excel файл
  9. Код УКТЗЕД так же загружается, но его нужно добавлять по другому
  10. Код УКТЗЕД добавляется в Tetra через Excel файл по шаблону, а с Tetra мигрирует в ETR 
  11. Код УКТЗЕД присылает поставщик в PIM core, но в ETR его загрузить нельзя
  12. Код Tetra попадает в PIM Core как то автоматически
  13. Есть сообщения об ошибках при передачи кода Tetra в PIM Core
  14. Колонка "Vendor Item No" = код Tetra для объединения вариантов (группировке штрих-кодов) в одну карточку товара 
  15. Колонка "GoodID Tetra" = уникальный код Tetra
  16. В Brocard.ua данные попадают с Navision
  17. В Letu.ua тоже передается заявка с 69 атрибутами товара
  18. Список полей, что передаются на Brocard.ua есть в файле "Поля по всім програмам.xlsx"
  19. В Экспорт с PIM core по штрих - коду подтягиваем код Tetra и формируем файл (вкладка НАВ "(імпорт осн.атриб)")
  20. При импорте в Navision добавляются поля с вкладка НАВ "(імпорт осн.атриб)". Остальные поля загружаются доп. файлами.(Структура файла: штрих - код, дополнительный атрибут)


Вопросы к товароведам:

  1. Все ли поля обязательные для заполнения при подгрузке файла в PIM Core? - готовится файл "Защита от дурака"
  2. Номенклатурные номера у различных марок могут повторяться? - Нет, не могут
  3. Спросить у Олега как данные мигрируют с ETR в Tetra 
  4. Увидеть файл загрузки в ETR (с учетом номенклатурного номера) - Да
  5. Увидеть файл выгрузки с ETR  не нужен, нужен только с Тетры
  6. Увидеть файл загрузки в Tetra с учетом поля "Область применения" _ Да
  7. Как добавить код УКТЗЕД? - проговорили
  8. Как код Tetra мигрирует в ETR?
  9. Обсудить поля, что есть в файле от Олега и заводятся в Tetra
  10. Как код Tetra попадает в PIM Core?
  11. Есть штрих - код, а есть штрих - код варианта. Как вы работает с вариантами в PIM core, ETR, Tetra(штрих - код будет же один). Штрих код варианта - это штрих код существующей позиции, если например поменялся объем фасовки
  12. Файлы, что импортируют дополнительные атрибуты - вышлют
  13. Если на один и тот же штрих-код будут разные атрибуты при импорте в Navision. Такая ситуация может быть? - Штрих-код уникальный
  14. Как вы редактируете текущий ассортимент? - вручную, по позиционно
  15. Удаляете ли вы позиции с ассортимента? - меняем статусы. В Navision архивные, новые
  16. Работа с Letu.ua (Кристина Кислова) Удалось пообщаться и уточнить детали?  - Нужно проговорить отдельно
  17. Процесс заведения наборов в ассортимент. В раскладке PIM core есть поле "Состав набора"
