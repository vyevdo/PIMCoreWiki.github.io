Цели

  • Оптимизировать процесс создания наборов

As it

  1. Завести набор как обычный товар в ETR, указав "тип товара" = Набор
  2. Данные с ETR переходят в Tetra office (В названии указывается в скобках состав набора, а также в объеме указываются объемы составляющих набора)
  3. Импорт в Navision делаем через excel file
    1. код товара Тетра
    2. ШК
    3. ТМ
    4. Линия
    5. назначение
    6. тип товара
    7. артикул
    8. код УКТ ЗЕД
  4. В Navision наполняем набор(существующими товарами либо создаем пустышки)

Предположения

  • Данные о составляющих набора обратно в Tetra office не транслируются
  • При создании составляющих набора можно указывать их количество
  • Атрибуты составляющих набора ничем не отличаются от обычного товара


To be

  1. Создать возможность для заведения набора с учетом составляющих в PIM Core
  2. Настроить импорт данной информации в Navision
